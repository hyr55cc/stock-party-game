# 📈 بورصة الأصدقاء

لعبة محفظة أسهم جماعية لـ 2-10 لاعبين، بتصميم فينتك عصري (على طراز Robinhood) مع رسوم بيانية حية لكل سهم. شاشة التلفزيون تعرض السوق والمتصدرين، وكل لاعب يتداول من جواله. تدعم الآن إعادة الاتصال التلقائي (إذا انقطع نت اللاعب أو المضيف، محفظتهم تبقى محفوظة عند العودة).

## طريقة التشغيل

يتطلب [Node.js](https://nodejs.org) مثبت على الجهاز (اللابتوب) الذي سيوصَل بالتلفزيون.

1. افتح الطرفية (Terminal / cmd) داخل مجلد `stock-party-game`.
2. ثبّت الحزم:
   ```
   npm install
   ```
3. شغّل السيرفر:
   ```
   npm start
   ```
4. في الطرفية ستظهر روابط، مثال:
   ```
   افتح شاشة التلفزيون على:
      http://localhost:3000/host.html
      http://192.168.1.5:3000/host.html
   واللاعبون يفتحون من جوالاتهم:
      http://192.168.1.5:3000/player.html
   ```
5. افتح رابط **host.html** على متصفح الجهاز الموصول بالتلفزيون (أو شارك الشاشة).
6. كل لاعب يفتح رابط **player.html** من جواله (بنفس شبكة الواي فاي)، أو يمسح رمز QR الظاهر على شاشة التلفزيون — يكتب اسمه وكود اللعبة المكوّن من 4 أرقام.
7. المضيف يختار مدة اللعبة (30 / 45 / 60 دقيقة) ويضغط "ابدأ اللعبة".

> **ملاحظة:** جميع الأجهزة (اللابتوب + جوالات اللاعبين) يجب أن تكون على نفس شبكة الواي فاي.

## رفع اللعبة أونلاين (يلعبها أي شخص من أي مكان)

نستخدم منصة **Render** لأنها مجانية بالكامل ولا تطلب بطاقة بنكية، وتدعم الاتصال المباشر (WebSocket) اللي تحتاجه اللعبة.

### 1) ارفع الكود إلى GitHub
1. اذهب إلى [github.com/new](https://github.com/new) وأنشئ مستودع (Repository) جديد باسم `stock-party-game` (خليه Public أو Private، ما يفرق).
2. داخل مجلد اللعبة على جهازك، افتح الطرفية وشغّل:
   ```
   git init
   git add .
   git commit -m "أول نسخة من بورصة الأصدقاء"
   git branch -M main
   git remote add origin https://github.com/USERNAME/stock-party-game.git
   git push -u origin main
   ```
   (بدّل `USERNAME` باسم حسابك على GitHub. إذا طلب منك تسجيل دخول، اتبع التعليمات على الشاشة.)

### 2) انشر على Render
1. روح إلى [render.com](https://render.com) وسجّل دخول بحساب GitHub.
2. اضغط **New +** → **Web Service**.
3. اختر مستودع `stock-party-game` من القائمة (اربط GitHub إذا طلب منك).
4. الإعدادات:
   - **Name**: أي اسم تحبه (مثلاً `stock-party-game`)
   - **Region**: أقرب منطقة لك
   - **Branch**: `main`
   - **Build Command**: `npm install`
   - **Start Command**: `npm start`
   - **Instance Type**: **Free**
5. اضغط **Create Web Service** وانتظر دقيقة أو اثنتين حتى ينتهي النشر.
6. بعد الانتهاء بيعطيك Render رابط دائم شبيه بـ:
   ```
   https://stock-party-game.onrender.com
   ```
   هذا الرابط يفتح شاشة المضيف تلقائياً. رابط اللاعبين هو نفسه بإضافة `/player.html`.

### ملاحظات مهمة عن النسخة المجانية من Render
- إذا ما فيه أحد يستخدم اللعبة لمدة 15 دقيقة، السيرفر "ينام" تلقائياً، وأول شخص يفتح الرابط بعدها ينتظر 30-60 ثانية لين يصحى. أثناء اللعب الفعلي ما راح تلاحظ أي بطء لأن فيه حركة بيانات مستمرة.
- بيانات اللعبة (اللاعبين، الأسعار) تُحفظ في ذاكرة السيرفر فقط، فإذا أعاد Render تشغيل السيرفر بيضيع تقدم اللعبة الحالية — يفضل تلعبون الجولة كاملة بجلسة واحدة.
- كل تحديث ترفعه لاحقاً على GitHub (`git push`) ينشره Render تلقائياً من جديد.
- إذا حاب أداء أسرع بدون نوم السيرفر، أرخص خيار هو ترقية الخطة داخل Render إلى **Starter** (مدفوعة شهرياً)، أو استخدام Railway.app كبديل (يتطلب بطاقة لكنه أسرع وما ينام).

## قواعد اللعبة

- كل لاعب يبدأ برصيد **100,000$** نقداً.
- 10 أصول للتداول: بيتكوين، إيثيريوم، ذهب، فضة، نفط، تسلا، آبل، أمازون، بنك الراجحي، إدريس.
- أسعار الأسهم تتحرك تلقائياً كل 11 ثانية (تقلبات عشوائية، كل سهم له درجة تقلب مختلفة — بعض الأسهم مثل بيتكوين وإيثيريوم وتسلا متقلبة بشكل جنوني)، وتُعرض كشموع يابانية مفرغة (hollow candles).
- التداول بالكمية (عدد الأسهم) وليس بالمبلغ — لكل سهم **سعر شراء** (أعلى قليلاً) و**سعر بيع** (أقل قليلاً)، والفرق بينهما هو "تكلفة" الصفقة بدل عمولة ثابتة. تُحسب لك تلقائياً **متوسط سعر الشراء** لكل سهم تملكه.
- سجل كامل لكل صفقاتك (شراء/بيع، الكمية، السعر، الوقت) متاح من زر "📜 الأوامر" في شاشة اللاعب.
- **تمويل مضاعف (Margin x2)**: يقدر كل لاعب يفعّل تمويلاً يضاعف رأس ماله النقدي مؤقتاً لمدة 5 دقائق (متاح 3 مرات كل لعبة). بعد انتهاء المهلة تُصفّى محفظته تلقائياً ويُسدد التمويل من الرصيد.
- أخبار مفاجئة (عامة أو خاصة بسهم واحد) تظهر كل 20-40 ثانية وتؤثر على الأسعار فوراً — إيجابية أو سلبية، وتظهر بشكل بارز مع صوت على شاشة التلفزيون، وكتنبيه على جوال كل لاعب.
- آبل وبنك الراجحي يوزعان أرباحاً نقدية دورية (كل 90 ثانية) لمن يملك أسهمهما.
- بطاقات فرصة/تحدي تُسحب تلقائياً كل 50 ثانية للاعب عشوائي: هبات نقدية، صفقة بدون فرق سعر، مضاعفة الأرباح، رسوم مفاجئة، أو تجميد سهم لمدة 30 ثانية.
- عند انتهاء الوقت، الفائز هو من يملك أعلى قيمة محفظة إجمالية (النقد + قيمة الأسهم بالسعر الحالي).

## أفكار لتوسيع اللعبة لاحقاً

- إضافة رافعة مالية (اقتراض لمضاعفة الصفقات مقابل مخاطرة أعلى).
- جولات "همس المطلعين" حيث يحصل لاعب عشوائي على تلميح مسبق بخبر قادم.
- وضع فرق (كل فريقين يشتركان في محفظة واحدة).
- متوسط سعر الشراء لكل مركز (Cost basis) في صفحة الأوامر.
